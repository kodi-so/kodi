# Kodi Brand Rollout Plan

Status: Active implementation plan
Prepared: April 3, 2026

This document breaks the proposed Kodi brand into milestones and ticket-sized work that can be created in Linear after approval.

## Project Name

Kodi Brand Rollout - Landing Page and Web App

## Success Criteria

- Kodi has one consistent visual and verbal identity across marketing and product.
- The attached logo mark replaces the temporary `K` badge in priority surfaces.
- ABeeZee is established as the primary brand typeface.
- The landing page communicates "conversation into completed work" more clearly than "generic AI assistant."
- The brand clearly explains Kodi's progressive autonomy model and user control.
- The app shell and key workflows feel like the same company as the landing page.

## Milestone 1: Brand Foundation

Goal:
Translate the approved brandbook into reusable assets, tokens, and rules.

Tickets:

1. Approve final Kodi brandbook and freeze canonical messaging
2. Create official Kodi logo asset pack from attached mark
3. Define shared color tokens for marketing and app surfaces
4. Add ABeeZee font loading and fallback strategy to `web` and `app`
5. Build reusable Kodi brand lockup component with mark + wordmark
6. Define approved autonomy language and delegation-state vocabulary
7. Document brand usage rules for engineering and design in repo

Exit criteria:

- approved brandbook
- reusable logo assets
- shared tokens documented
- brand lockup component ready for use

## Milestone 2: Landing Page Rebrand

Goal:
Make the marketing site clearly communicate Kodi's position as the AI teammate that turns meetings into execution and can carry work itself under user control.

Tickets:

1. Replace placeholder `K` branding with official Kodi mark on landing page
2. Update homepage headline and subhead to approved brand language
3. Rewrite value sections around decisions, execution, and follow-through
4. Add messaging that explains Kodi's progressive autonomy and user control
5. Rework startup and SMB proof points for setup, pricing, trust, and control
6. Update landing page typography to use ABeeZee-led hierarchy
7. Replace purple/indigo AI styling with Kodi palette
8. Add visual proof modules that show draft, approval, and direct execution states
9. QA landing page responsiveness, accessibility, and contrast after rebrand

Exit criteria:

- homepage reflects approved messaging
- brand visuals are consistent with the new system
- responsive and accessibility checks pass

## Milestone 3: Web App Shell Rebrand

Goal:
Bring the core application shell into alignment with the approved Kodi brand.

Tickets:

1. Replace sidebar, auth, and onboarding `K` badge with official Kodi mark
2. Rebrand app shell color tokens around Room Dark, Kodi Ink, and Signal Gold
3. Update auth pages to match the new typography and palette
4. Rebrand sidebar, nav states, and account surfaces to new token system
5. Update shared cards, buttons, badges, and empty states to brand rules
6. Align app metadata, page titles, and top-level product copy with brand voice

Exit criteria:

- auth and app shell feel like the same brand as marketing
- placeholder identity elements removed from top-level app surfaces

## Milestone 4: Workflow + Autonomy Surface Rebrand

Goal:
Apply the brand to the most important product experiences where trust and clarity matter.

Tickets:

1. Rebrand meetings list and meeting detail surfaces
2. Rebrand chat interface assistant visuals and delegation language
3. Rebrand integrations catalog and integration detail flows around connected-account execution
4. Rebrand approvals workflow to emphasize control and trust
5. Rebrand autonomy and policy settings to reflect the delegation spectrum
6. Rebrand settings and workspace management surfaces
7. Standardize loading, live, success, warning, error, and autonomy states across workflows

Exit criteria:

- high-traffic product pages share a consistent brand language
- system states read clearly and feel intentional

## Milestone 5: Launch Readiness

Goal:
Ship the rebrand cleanly and support go-to-market usage.

Tickets:

1. Capture updated marketing and product screenshots for future sales use
2. Run final copy QA against approved voice and terminology
3. Run final accessibility and responsive audit across landing and app
4. Create a brand-release checklist for launch day
5. Prepare announcement copy for website, email, and social
6. Define post-launch metrics for trial conversion and activation impact

Exit criteria:

- assets ready for launch
- quality bar confirmed
- launch messaging prepared

## Suggested Ticket Order

1. Milestone 1 first
2. Milestone 2 and Milestone 3 can overlap once foundation work is ready
3. Milestone 4 follows shared shell and token work
4. Milestone 5 closes the project
